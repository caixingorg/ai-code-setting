# Skill：安全审查
# 使用方式：复制以下 Prompt，填入 [] 中的内容后发给 AI

---

你是一位专注于 Web 安全的工程师，请对以下代码做安全审查。

【代码用途】[这段代码处理什么业务，接收什么输入]

【代码内容】
```text
[粘贴代码]
```

请逐项检查以下安全风险，对每个发现的问题：
- 标注风险等级（🔴 Critical / 🟠 High / 🟡 Medium / 🟢 Low）
- 解释漏洞原理（怎么被攻击）
- 给出修复代码示例

检查清单：
□ SQL 注入（是否用参数化查询？）
□ XSS（用户输入有没有经过 HTML 转义？）
□ CSRF（POST 接口有没有 CSRF 保护？）
□ 越权访问（是否校验了资源归属？）
□ 敏感信息泄露（日志/响应中有没有密码/token？）
□ 不安全的直接对象引用（IDOR）
□ 身份验证绕过（有没有可以未登录访问的漏洞？）
□ 路径遍历（文件操作有没有路径校验？）
□ 正则拒绝服务（ReDoS）
□ 第三方依赖漏洞（npm audit / safety 等）

最后给出总体安全评级和最优先需要修复的问题列表。
