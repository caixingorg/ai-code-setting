# Skill：代码 Review
# 使用方式：复制以下 Prompt，填入 [] 中的内容后发给 AI

---

你是本项目的技术负责人，正在 Review 以下代码。

【PR/变更背景】
[简述这次改动的目的：修复了什么 Bug / 实现了什么功能]

【需要 Review 的代码】
```text
[粘贴代码]
```

请从以下维度逐项 Review，每个问题按严重程度标注（🔴 必须修改 / 🟡 建议修改 / 🟢 可以考虑）：

**正确性**
- 逻辑是否正确？有没有边界条件遗漏？
- 并发场景下是否安全？

**安全性**
- 是否有注入风险（SQL / XSS / 命令注入）？
- 用户输入是否经过校验和转义？
- 是否有越权访问漏洞？

**性能**
- 是否有 N+1 查询？
- 有没有不必要的重复计算或大量内存分配？
- 循环中有没有隐藏的 O(n²) 操作？

**可维护性**
- 函数是否太长（超过 50 行需要考虑拆分）？
- 命名是否清晰准确？
- 有没有重复代码可以抽取？

**测试覆盖**
- 核心逻辑是否有对应测试？
- 测试是否覆盖了异常路径？

最后给出总结：是否可以直接合并？还是需要修改后再看？
