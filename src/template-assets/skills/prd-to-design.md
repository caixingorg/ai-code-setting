# Skill：需求转技术方案
# 使用方式：复制以下 Prompt，填入 [] 中的内容后发给 AI

---

你是本项目的技术负责人。我有一个产品需求，请帮我输出完整技术方案。

【需求来源】[PRD v1.2 / 产品会议记录 / 口头需求]

【需求描述】
[完整粘贴 PRD 或需求描述，越详细越好]

【背景约束】
- 上线时间要求：[X 周后]
- 参与开发人数：[X 人]
- 是否需要兼容旧数据：[是/否，如是请说明]

请按以下格式输出技术方案：

**1. 需求理解**
用技术语言复述需求（确认我们对需求理解一致）。
列出所有模糊点和待产品确认的问题。
列出隐含需求（产品没说但技术上必须考虑的点）。

**2. 数据库变更**
需要新增/修改的表和字段。
给出具体的 Schema 定义（Prisma / SQL DDL）。

**3. API 接口设计**
列出所有需要新建/修改的接口。
每个接口：路径、方法、入参、出参、权限要求。

**4. 业务逻辑**
核心流程用 ASCII 流程图或步骤列表描述。
重点标注：事务边界、并发风险点、外部依赖。

**5. 前端组件拆解**（如涉及）
需要新建/修改的组件树。
哪些是纯展示组件，哪些需要状态管理。

**6. 风险点**
性能风险（高并发、大数据量场景）。
安全风险（权限、数据隔离）。
兼容性风险（旧数据、旧客户端）。

**7. 开发任务拆分**
按「从底层到上层」排序，标注依赖关系。
每个任务格式：[ ] 任务名 | 依赖：xxx | 估时：xh
