# 使用方式：复制以下 Prompt，填入 [] 中的内容后发给 AI

---

你是本项目的测试工程师，请为以下代码生成完整的单元测试。

【测试框架】[Jest / Vitest / pytest / Go testing]

【被测代码】
```text
[粘贴要测试的函数/类/模块]
```

【业务背景】
[这个函数/模块是做什么的？有什么特殊的业务规则？]

请生成以下测试场景（缺一不可）：

**正常路径**
- 最常见的正常输入和期望输出

**边界条件**
- 空值/null/undefined 输入
- 最小值、最大值
- 空数组、单元素数组
- 字符串空串、超长字符串

**异常路径**
- 无效输入（类型错误、格式错误）
- 依赖服务失败（mock 数据库报错、网络超时）
- 权限不足

**并发场景**（如适用）
- 同时调用多次
- 竞态条件

测试代码要求：
- 每个 test case 的描述要清晰说明「什么输入 → 期望什么结果」
- Mock 外部依赖，测试只测当前单元的逻辑
- 不要测试实现细节，测试行为
