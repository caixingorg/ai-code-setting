# Agent 自主执行规范 Rules
# 当 AI 以 Agent 模式自主执行任务时，必须遵守以下规则

## 执行前
- 任何超过 3 个文件的改动，必须先输出完整计划（文件清单 + 改动说明）
- 有破坏性操作（删除、覆盖、数据库变更）时，必须明确提示并等待确认
- 执行前确认当前 Git 分支，禁止在 main/master 上直接执行

## 执行中
- 每完成一个逻辑单元，输出进度摘要
- 遇到不确定的逻辑（如业务规则模糊），停下来提问而不是自行假设
- 如果发现执行计划有误，立刻停止并重新报告，不要将错就错

## 执行后
- 列出所有被创建/修改/删除的文件
- 说明是否需要运行 migration、install 依赖等后续步骤
- 主动指出本次修改可能的副作用或需要人工验证的地方

## 绝对禁止
- 修改 .env、.env.local、.env.production 等环境变量文件
- 提交代码到 Git（除非明确被要求）
- 执行 npm publish、docker push 等发布操作
- 访问项目目录之外的文件系统
