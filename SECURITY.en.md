# Security Policy

Thanks for helping improve the security of this project.

[中文](SECURITY.md) | [English](SECURITY.en.md)

## Supported Scope

The latest code on the `main` branch is the default supported target.

## How to Report

Before a dedicated security contact is configured for the repository:

- Do not publicly disclose exploitable vulnerability details
- Do not include secrets, tokens, credentials, or attack payloads in public issues
- Prefer the hosting platform's private security reporting feature when available

If private reporting is not enabled yet, wait for the maintainer to publish a secure contact path before sending full details.

## Please Include

- Affected version or commit range
- Vulnerability type and impact
- Minimal reproduction steps
- Possible remediation direction
- Whether sensitive data exposure is involved

## Response Expectations

The maintainer will review reports as soon as practical, assess impact, and plan a fix. Public disclosure should wait until a mitigation or fix is available.
