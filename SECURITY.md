# 安全策略

感谢你帮助提升项目安全性。

## 支持范围

当前默认关注 `main` 分支上的最新代码。

## 报告方式

在仓库尚未配置专用安全联系渠道前：

- 不要公开提交可被直接利用的漏洞细节
- 不要在公开 issue 中附带密钥、令牌、凭证或可复现攻击载荷
- 优先使用仓库平台提供的私密安全报告能力

如果当前平台尚未启用私密报告能力，请等待维护者补充安全联络方式后再提交完整细节。

## 报告建议包含

- 受影响版本或提交范围
- 漏洞类型与影响面
- 最小复现步骤
- 可能的修复方向
- 是否涉及敏感数据泄露

## 响应预期

维护者会在条件允许时尽快确认问题、评估影响并安排修复。公开披露应在修复方案可用后进行。
